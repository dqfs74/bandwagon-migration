## 换机房之前，必须先搞清楚这几件事

### 换完机房，IP 地址会变

这是最重要的一点，没有之一。

迁移到新机房之后，搬瓦工会给你分配一个新的 IPv4 地址。如果你的 VPS 上跑着网站，绑着域名解析，换完要立刻去更新 DNS 记录，否则网站会断。做过一次就知道，忘了改 DNS 是血泪教训。

顺便提一嘴：如果你之前开启了 IPv6，切换机房后 IPv6 地址变化不确定，用之前最好重新删掉再加，延迟会更好。

### 换机房会消耗流量

不是免费无损的，这里有个细节：迁移过程中，系统需要把你整个磁盘镜像传到新机房，会消耗你的月度流量额度。VPS 里数据越多，消耗越多，一般是已用磁盘空间的好几倍。

想省流量？迁移前先清理一下用不到的大文件，或者建完快照再删。

### IP 被封的情况下，不能换机房

这是个非常常见的误解——以为 IP 被墙了可以通过换机房来绕过去。错。

搬瓦工的规定是：**IP 被封之后，"Migrate to another DC"功能会被锁住，没法用**。这时候唯一的办法是先付费购买一个新 IP（目前价格是 $7.39），恢复正常 IP 之后，才能走免费的机房迁移流程。所以 IP 被封和换机房是两件事，别混。

### 不同套餐，能换的机房不一样

KVM 普通套餐、CN2 GIA-E 套餐、香港套餐，可选机房的列表完全不同。

大致规则是：低级别机房不能迁往高级别。香港 CN2 GIA 是独立套餐，只能单独购买，其他套餐迁不进去。CN2 GIA-E 套餐是可选机房最多的，目前支持 12 个以上机房自由切换，包括洛杉矶 DC6、DC9、日本软银、荷兰联通 EUNL_9 等。

## 搬瓦工如何更改机房：一步一步来

### 第一步：进入 KiwiVM 控制面板

登录搬瓦工账号之后，进入 "Services" → "My Services"，找到你的 VPS 实例，点击右侧的小齿轮图标 ⚙️，选择 "Open KiwiVM"，就能进入该 VPS 的后台控制面板。

### 第二步：找到机房迁移功能

KiwiVM 左侧菜单往下拉，找到 "Migrate to another DC" 这个选项，点进去。

右侧会显示两个信息：当前机房（Current location of this VPS）和可以迁移的目标机房列表。如果这个入口点进去显示的是 "Plan does not support migrations"，说明你这个套餐本身不支持机房切换，需要换一个套餐。

### 第三步：选择目标机房，确认迁移

从列表里选一个你想换过去的机房，点 "Confirm on next step"，然后再次确认，点 "Start Migration"，迁移就开始了。

迁移期间 VPS 还可以操作，到最后阶段会有几秒到几分钟的短暂关机。整体完成时间取决于你 VPS 里的数据量，一般半小时以内。迁移完成后会发一封邮件到你的注册邮箱，里面有新的 IP 地址。

### 第四步：检查迁移结果

收到邮件之后，用新 IP 登录 SSH 确认一下环境，服务还在、数据还在，一切正常。

如果你的 VPS 上有域名绑定，这时候同步去 DNS 控制台更新 A 记录，把旧 IP 改成新 IP，传播一般 5 分钟到 1 小时不等。

## 换机房常见问题（FAQ）

**Q：搬瓦工更改机房是否收费？**

不收费，次数也没有限制，可以随时切换。但每次迁移会消耗月度流量，要注意剩余流量是否够用。

**Q：换机房之后，VPS 里的数据会丢吗？**

不会丢。迁移过程是把整个磁盘镜像传输到新机房，数据全部保留，只是 IP 地址会变。尽管如此，迁移前建个快照是好习惯，以防万一。

**Q：我的 IP 被墙了，换机房能解决吗？**

不能。IP 被封之后，机房迁移功能会被锁定，无法使用。需要先在 KiwiVM 后台付费购买新 IP（目前 $7.39），IP 恢复正常之后才能换机房。

**Q：KVM 套餐可以换到 CN2 GIA 机房吗？**

可以，但换过去之后月流量会变成原套餐的三分之一。比如原来有 1TB 流量，换到 CN2 机房就只剩约 333GB。这个规则在选机房之前要先算清楚。

**Q：迁移要多久，期间 VPS 能用吗？**

迁移过程中 VPS 基本还可以正常操作，到最后阶段会短暂关机几秒到几分钟。整体耗时看数据量，VPS 里数据少的话十几分钟可能就搞定了，多的话不超过半小时是常规情况。迁移完成后会收到邮件通知。

**Q：香港机房可以通过迁移换过去吗？**

不可以。香港 CN2 GIA 是独立套餐，其他套餐没法迁移进去，必须单独购买香港套餐。
